# MVP選出ロジック

## 概要

MVP選出は **Gemini AI による総合評価** で行います。
試合データ（得点・勝利）、メンバー情報（経験・年次・備考・幹事）、アンケートコメントの全てをAIに渡し、**0〜100点** で各メンバーを採点します。

選出対象は **ラウンド（マッチ）に1回以上出場したメンバーのみ** です。

**前提条件**: MVP選出はイベントのステータスが「試合終了」の場合のみ実行可能です。「進行中」や「完了」では実行できません。

## 設計思想

社内・仲間内のフットサルイベントでは、得点や勝利数だけでMVPを決めると、サッカー経験者が常に上位になりがちです。
このシステムでは、得点などの定量データはあくまで参考情報としてAIに渡し、チームメイトからの評価や場の雰囲気への貢献を重視した、現場感のあるMVP選出を目指しています。

## 処理フロー

```
1. データ取得
   イベントに紐づくメンバー・ラウンド・マッチ・得点・アンケート回答を一括取得

2. 参加メンバー抽出
   マッチメンバーテーブルから、対象マッチに出場したメンバーIDを重複なしで抽出

3. 試合データ集計
   各メンバーの得点数・勝利数・出場試合数を集計（AI入力用）

4. Gemini AI 総合評価（1回のAPI呼び出し）
   全データをプロンプトに含めてAIに送信し、以下を一括で取得:
   - 各メンバーの総合スコア（0〜100点）
   - MVP / 準MVP の順位付け
   - 全メンバーの評価コメント

5. 結果保存
   既存のMVP結果を削除し、新しい結果をスプレッドシートに書き込み
```

## AIに渡す情報

各メンバーについて、以下の情報をプロンプトに含めます。

| 情報 | 取得元 | 説明 |
|------|--------|------|
| 名前 | メンバーテーブル | メンバーの名前 |
| サッカー経験 | メンバーテーブル | あり / なし |
| 年次 | メンバーテーブル | 1〜 |
| 備考 | メンバーテーブル | 初参加、久しぶりなどの自由入力テキスト |
| 幹事 | メンバーテーブル | はい / いいえ |
| 出場試合数 | マッチメンバーから集計 | 出場したマッチの数 |
| 得点数 | 得点テーブルから集計 | 対象マッチでの個人得点合計 |
| 勝利数 | マッチ結果から集計 | 所属チームが勝利したマッチの数 |
| コメント | アンケート回答テーブル | チームメイトからの評価コメント |

## AIへの評価方針

プロンプトに含まれる評価指示は以下の通りです。

- 社内・仲間内のフットサルイベントであり、プロの試合ではない
- 得点や勝利数だけでなく、現場の雰囲気への貢献・チームメイトからの評価を重視する
- サッカー未経験者や若手が積極的に参加し頑張っている場合は高く評価する
- 備考欄の情報（初参加、久しぶりの参加など）も考慮する
- 幹事としてイベント運営に貢献しているメンバーも評価に加味する
- チームメイトからのコメントが多い・ポジティブなメンバーは高く評価する
- 得点が多い＝MVPではない。場を盛り上げた人、チームに貢献した人を総合的に評価する

### 採点基準の目安

| スコア | 評価 |
|--------|------|
| 90〜100点 | 文句なしのMVP。試合でも雰囲気でも圧倒的な貢献 |
| 70〜89点 | MVP候補。目立った活躍や周囲からの高い評価 |
| 50〜69点 | 良い活躍。チームに貢献していた |
| 30〜49点 | 普通の参加。特筆すべき点は少ない |
| 0〜29点 | 参加のみ。目立った活躍やコメントがない |

## AIの出力形式

AIは以下のJSON配列を返します。

```json
[
  {"memberId": "xxx", "score": 85, "rank": "MVP", "reason": "評価コメント"},
  {"memberId": "yyy", "score": 72, "rank": "準MVP", "reason": "評価コメント"},
  {"memberId": "zzz", "score": 60, "rank": "", "reason": "評価コメント"}
]
```

- `score`: 0〜100の整数（範囲外はクランプ）
- `rank`: `MVP` / `準MVP` / `""` のいずれか
- `reason`: 50〜80文字程度の評価コメント

## 順位付け

MVP人数・準MVP人数はイベント設定で指定します（各1〜5人、デフォルト各1人）。
AIがスコアと順位を同時に決定するため、ルールベースの順位割り当ては行いません。

## 特殊ケース

| ケース | 挙動 |
|--------|------|
| 参加メンバーが0人 | エラーを返す（「参加メンバーがいません」） |
| アンケート回答が0件 | コメントなしの状態でAI評価を実行（得点・勝利・メンバー情報のみで判断） |
| AI API呼び出し失敗 | エラーを返す（API KEYの設定確認を促す） |
| AIレスポンスの解析失敗 | エラーを返す |
| AIが返したスコアの範囲外 | 0〜100にクランプし、整数に丸める |
| AIがメンバーを返さなかった場合 | スコア0、順位なし、コメントなしで補完 |

## 結果の保存

MVP結果テーブル（スプレッドシート）に以下のカラムで保存されます。

| カラム | 内容 |
|--------|------|
| イベントID | 対象イベントのID |
| メンバーID | メンバーのID |
| 名前 | メンバーの名前 |
| 順位 | `MVP` / `準MVP` / 空文字 |
| 理由 | AIが生成した評価コメント |
| 総合スコア | AIによる総合スコア（0〜100点） |

再選出時は既存のMVP結果を **全削除してから再作成** します（冪等性を確保）。
AIによる評価のため、再選出のたびに結果が変わる可能性があります。

**注意**: MVP選出を実行してもイベントのステータスは自動で変更されません。イベントを完了にするには、管理者が明示的に「完了にする」ボタンを押す必要があります。
