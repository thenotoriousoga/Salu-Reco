# Futsal Manager — デザインシステム

## 概要

Futsal Manager は、社内・仲間内のフットサルイベントをスマホブラウザから管理するWebアプリです。
Google Apps Script（GAS）上で動作するSPA構成で、試合のリアルタイムスコア管理からMVP選出まで、イベント当日の現場で使われることを前提に設計しています。

### デザインコンセプト

**Vibrant & Block-based** — スポーツの躍動感を表現する鮮やかな配色と、情報を明確に区切るブロックレイアウト。
体育館やフットサルコートの照明下でも視認性が高く、片手操作でストレスなく使えることを最優先にしています。

### デザイン原則

| 原則 | 説明 |
|------|------|
| **スマホファースト** | 最大幅600pxのコンテナ。片手の親指で届く範囲にアクションを配置 |
| **即座のフィードバック** | すべての操作にローディング・トースト・アニメーションで応答 |
| **ロール別の情報設計** | 管理者と一般ユーザーで見える情報を明確に分離 |
| **現場で迷わない** | 初見でも操作手順がわかるステップ表示と空状態ガイド |

## コンポーネント

### ボタン

4つのバリエーションと3つのサイズを組み合わせて使用。

| バリエーション | クラス | 背景 | テキスト | 用途 |
|----------------|--------|------|----------|------|
| Primary | `.btn-primary` | `--primary` | 白 | メインアクション（参加、登録、試合開始） |
| Secondary | `.btn-secondary` | `--border-light` | `--text` | 補助アクション（戻る、全選択） |
| Accent | `.btn-accent` | `--accent` | `--text` | 特別なアクション（AI分け） |
| Danger | `.btn-danger` | 透明 + 赤枠 | `--danger` | 破壊的操作（削除） |

| サイズ | クラス | パディング | 特徴 |
|--------|--------|------------|------|
| Large | `.btn-large` | 14px 20px | 幅100%、主要アクション |
| Default | `.btn` | 10px 20px | インライン配置 |
| Small | `.btn-sm` | 8px 14px | 最小44x44px（タッチターゲット確保） |

**インタラクション:**
- ホバー: 背景色が暗くなり、シャドウが強まる
- アクティブ: `scale(0.97)` で押し込み感
- フォーカス: `outline: 3px solid var(--primary)` + `offset: 2px`
- トランジション: 200ms ease

### カード

情報のグルーピングに使用する基本コンテナ。

```
.card {
  background: var(--bg-card);      /* 白背景 */
  border-radius: var(--radius);    /* 16px */
  padding: 20px;
  margin-bottom: 16px;
  box-shadow: var(--shadow-md);
  border: 1px solid var(--border-light);
}
```

**アニメーション:** `prefers-reduced-motion: no-preference` の場合、`fadeUp`（8px上方向から300msでフェードイン）

### 入力フィールド

```
.input {
  padding: 10px 14px;
  border: 2px solid var(--border);
  border-radius: var(--radius-sm);  /* 10px */
  appearance: none;                 /* ネイティブスタイルをリセット */
}
```

**フォーカス:** ボーダーが `--primary` に変わり、`box-shadow: 0 0 0 3px rgba(220,38,38,0.15)` のリングが表示

### タブバー

3タブ構成（メンバー / 試合 / 結果）。

- 背景: 白カード風
- アクティブタブ: `--primary` 色のテキスト + 下線（3px）
- 角丸: `--radius`（16px）で丸みのあるピル型

### スコアボード

試合中のスコア表示。中央にVSバッジ、左右にチーム名・スコア・+1/-1ボタンを配置。

- スコア数字: Fira Code 3.5rem 700
- VSバッジ: 40x40px 円形、白背景 + ボーダー
- 背景: `linear-gradient(135deg, var(--bg-surface), var(--border-light))`
- **スコアバウンスアニメーション:** 得点変更時に `scale(1.2)` → `scale(1)` の300msバウンス

### トースト通知

画面下部中央に表示されるフィードバック。

- 位置: `bottom: 80px`、中央寄せ
- 背景: `--text`（通常）/ `--danger`（エラー）
- アニメーション: 下から300msでスライドイン、2.5秒後に自動消去
- ARIA: `role="alert"` + `aria-live="polite"`

### カスタムモーダル

`confirm()` の代替として実装。デザインシステムに統一されたスタイル。

- オーバーレイ: `rgba(127,29,29,0.4)` + `backdrop-filter: blur(4px)`
- ボックス: 白背景、最大幅360px、`--shadow-lg`
- ARIA: `role="dialog"` + `aria-modal="true"` + `aria-labelledby`
- 操作: オーバーレイクリックで閉じる、確認ボタンに自動フォーカス

### ローディング

フルスクリーンオーバーレイ + スピナー。長時間処理時はメッセージ表示に対応。

- オーバーレイ: `rgba(254,242,242,0.85)` + `backdrop-filter: blur(4px)`
- スピナー: 44x44px、`--primary` 色の回転アニメーション（0.7s）
- メッセージ: タイトル（600ウェイト）+ サブテキスト（0.8rem、muted色）
- ARIA: `role="status"` + `aria-label="読み込み中"`

### ステップインジケーター

ラウンド作成の3ステップを視覚化。

| ステップ | 状態 | スタイル |
|----------|------|----------|
| ① メンバー選択 | active | `--primary` 背景、白テキスト |
| ② チーム分け | active | 同上 |
| ③ 試合開始 | active | 同上 |
| 完了済み | done | `--success-bg` 背景、`--success` テキスト |
| 未到達 | default | `--bg-surface` 背景、`--text-muted` テキスト |

### 空状態（Empty State）

データがない画面に表示するガイド。

- SVGアイコン: 48x48px、`--border` 色、`stroke-width: 1.5`
- メインテキスト: 0.9rem、`--text-muted`
- サブテキスト: 0.8rem、`--text-muted`、次のアクションを案内

### バッジ

メンバー属性やステータスを小さなラベルで表示。

## アイコンシステム

### 方針

- **SVGインライン** を使用（外部ライブラリ不使用）
- viewBox: `0 0 24 24`
- stroke-based: `stroke-width: 2`, `stroke-linecap: round`, `stroke-linejoin: round`, `fill: none`
- すべてのアイコンに `aria-hidden="true"` を付与

### アイコン管理

JSの `svg()` ヘルパー関数と `IC` オブジェクトで一元管理。

```javascript
// ヘルパー関数（aria-hidden自動付与）
function svg(paths, size) {
  size = size || 14;
  return '<svg aria-hidden="true" style="width:' + size + 'px;..." viewBox="0 0 24 24">' + paths + '</svg>';
}

// アイコン定義
var IC = {
  goal:   '<circle cx="12" cy="12" r="10"/>',
  trophy: '<path d="M6 9H4.5a2.5 2.5 0 0 1 0-5C7 4 9 7 12 7s5-3 7.5-3..."/>',
  check:  '<polyline points="20 6 9 17 4 12"/>',
  x:      '<line x1="18" y1="6" x2="6" y2="18"/><line x1="6" y1="6" x2="18" y2="18"/>',
  minus:  '<line x1="5" y1="12" x2="19" y2="12"/>',
  file:   '<path d="M14 2H6a2 2 0 0 0-2 2v16a2 2 0 0 0 2 2h12..."/>',
  trash:  '<polyline points="3 6 5 6 21 6"/><path d="M19 6v14a2 2 0 0 1-2 2H7..."/>',
  flag:   '<path d="M4 15s1-1 4-1 5 2 8 2 4-1 4-1V3s-1 1-4 1-5-2-8-2-4 1-4 1z"/>'
};
```

### サイズ規約

| 用途 | サイズ | 例 |
|------|--------|------|
| ヘッダーアイコン | 24px | ロゴ |
| セクションアイコン | 22px / 18px | カード見出し |

## アニメーション・トランジション

### トランジション

| 対象 | 時間 | イージング | プロパティ |
|------|------|------------|------------|
| 汎用 | 200ms | ease | `--transition` |
| トースト | 300ms | ease | transform, opacity |
| スピナー | 700ms | linear | transform (rotate) |

### アニメーション

| 名前 | 効果 | 条件 |
|------|------|------|
| `fadeUp` | 8px上方向からフェードイン（300ms） | カード表示時 |
| `scoreBounce` | `scale(1)` → `scale(1.2)` → `scale(1)`（300ms） | スコア変更時 |
| `spin` | 360度回転（700ms、無限ループ） | ローディングスピナー |

### モーション配慮

`@media (prefers-reduced-motion: no-preference)` でアニメーションを条件付き適用。
ユーザーがOSレベルで「視差効果を減らす」を有効にしている場合、`fadeUp` と `scoreBounce` は無効化されます。

## ロール別UI設計

CSSクラスベースの表示制御。JavaScriptで `body` に `role-admin` または `role-user` クラスを付与し、CSSで要素の表示/非表示を切り替えます。

```css
body.role-user .admin-only  { display: none !important; }
body.role-admin .user-only  { display: none !important; }
```

- イベント作成フォーム
- メンバー登録フォーム

- ラウンド削除ボタン
- イベントコード表示

- アンケートフォーム作成

## インタラクションパターン

### サーバー呼び出し（`callServer`）

すべての `google.script.run` 呼び出しを `callServer()` ラッパーで統一。

1. ローディングオーバーレイ表示
2. サーバー関数実行
3. 成功: ローディング非表示 → コールバック実行
4. 失敗: ローディング非表示 → エラートースト表示

### 楽観的UI更新（スコア・得点）

スコアの+1/-1操作は即座にUIを更新し、500msデバウンスでサーバーに保存。
ネットワーク遅延を感じさせない操作感を実現しつつ、連打時のAPI呼び出しを最小化。

### 確認ダイアログ

破壊的操作（削除、MVP選出）にはカスタムモーダルを表示。
ブラウザネイティブの `confirm()` ではなく、デザインシステムに統一されたスタイルで表示。

### キュー方式のメンバー登録

1. メンバー情報を入力 → 「リストに追加」で追加待ちキューに追加
2. キュー内のメンバーを確認・削除

## ファイル構成

| ファイル | 役割 |
|----------|------|
| `css.html` | デザイントークン（CSS変数）、全コンポーネントスタイル |
| `index.html` | HTML構造、ページレイアウト、ARIA属性 |
| `js.html` | コア機能（状態管理、ルーティング、ログイン、ユーティリティ） |
| `js-members.html` | メンバー管理UI（キュー、一覧、バッジ） |
| `js-rounds.html` | 試合管理UI（チーム分け、スコアボード、得点記録） |
| `js-results.html` | 結果表示UI（成績、アンケート、MVP） |

## 変更時のガイドライン

### 色を変更する場合

`:root` のCSS変数のみを変更してください。ハードコードされた色値は使用しないでください。
例外: MVP結果カードの金・銀背景（`#FFFBEB`, `#F8FAFC`）はCSS変数化されていません。

### コンポーネントを追加する場合

1. 既存のボタン・カード・バッジのバリエーションで対応できないか検討
2. 新しいCSSクラスは `css.html` のセクションコメント（`/* --- セクション名 --- */`）に従って配置
3. アイコンは `IC` オブジェクトに追加し、`svg()` ヘルパーで描画
4. 管理者限定の要素には `admin-only` クラスを付与

### アニメーションを追加する場合

必ず `@media (prefers-reduced-motion: no-preference)` で囲んでください。
トランジション時間は `--transition`（200ms）を基準に、300ms以下に収めてください。
