# 実装予定機能

## 概要

ユーザーフィードバックを基に選定した追加機能の一覧。
必要度・実装難易度・推奨実装順についてまとめる。

## 機能一覧

### 1. 得点ランキング / 勝ち点ランキング / スタッツ

| 項目 | 内容 |
|------|------|
| 必要度 | ★★★★★ |
| 実装難易度 | 中 |
| カテゴリ | データ・統計 |
| 実装順 | 1番目 |
| ステータス | 実装済み |

**概要**: 既存の「イベント内成績」セクションに、得点ランキング・勝ち点ランキング・スタッツ（AIレーティング）の切り替え機能を追加する。各カードはタップで展開し、試合別の結果とAI評価コメントを表示する。

#### 方針

- 新規セクションの追加ではなく、既存の `renderEventStats()` を拡張する
- スタッツ（AIレーティング）はMVP選出と同時に算出し、追加のAPI呼び出しは行わない
- イベント進行中はスタッツ列が非表示、MVP選出後に表示される

#### ランキングモード

| モード | ソート順 | 強調表示 | 表示タイミング |
|--------|----------|----------|---------------|
| 得点ランキング | 得点数 降順 → 勝利数 降順 | 得点数を目立たせる | 常時 |
| 勝ち点ランキング | 勝ち点 降順 → 得点数 降順 | 勝敗（勝ち点）を目立たせる | 常時 |
| スタッツ | レーティング 降順 | レーティングを目立たせる | MVP選出後のみ |

#### 勝ち点の計算

- 勝利: 3点
- 引き分け: 1点
- 敗北: 0点

#### スタッツ（AIレーティング）

- Sofascore風の10段階評価（0.0〜10.0、小数第一位まで）
- MVP選出時にGemini APIで全員分を一括算出
- 算出に使うデータ: 試合データ（得点・勝敗・出場数）+ メンバー情報（経験・年次・備考・幹事）+ アンケートコメント
- MVP選出の既存プロンプトを拡張し、1回のAPI呼び出しで「MVP結果 + 全員のレーティング + 評価コメント」を返す

#### タップ展開（個人詳細）

各ランキングカードをタップすると、以下の詳細情報がアコーディオン展開される:

| 項目 | 内容 |
|------|------|
| 試合別結果 | 各試合の勝敗・得点を一覧表示 |
| AI評価コメント | Gemini APIが生成した個人への評価コメント（MVP選出時に算出） |

- 試合別結果はクライアント側の `cachedRounds` から取得（追加API不要）
- AI評価コメントはMVP選出後のみ表示（未選出時は試合別結果のみ展開）

#### 切り替えUI

- **セグメントボタン**（ピル型の切り替えボタン）を採用
- 「イベント内成績」見出しの直下に配置
- イベント進行中: `[⚽ 得点]` `[🏆 勝ち点]` の2ボタン
- MVP選出後: `[⚽ 得点]` `[🏆 勝ち点]` `[📊 スタッツ]` の3ボタン
- 選択中のボタンがハイライトされる
- 既存のタブバー（メンバー/試合/結果）と視覚的に混同しないデザイン

#### MVP選出フローの変更

```
現状:
  管理者が「MVPを発表」→ Gemini API → MVP/準MVP + 順位 + 理由 + 総合スコア

変更後:
  管理者が「MVPを発表」→ Gemini API → MVP/準MVP + 順位 + 理由 + 総合スコア
                                       + 全員のレーティング(0.0〜10.0)
                                       + 全員の評価コメント
```

#### データ保存

- MVP結果シートにカラムを追加: `レーティング`（数値）、`評価コメント`（文字列）
- MVP/準MVPに選ばれなかったメンバーも含め、全員分のレーティング・評価コメントを保存

#### 影響範囲

| ファイル | 変更内容 |
|----------|----------|
| `src/js-results.html` | `renderEventStats()` の拡張（3モード切り替え、タップ展開、スタッツ表示） |
| `src/index.html` | セグメントボタンのHTML追加（`#event-stats` 周辺） |
| `src/css.html` | セグメントボタン・アコーディオン展開・レーティング表示のスタイル追加 |
| `src/Mvp.gs` | Geminiプロンプト拡張（全員のレーティング・評価コメントを要求） |
| `src/Code.gs` | MVP結果シートのヘッダーに `レーティング`・`評価コメント` カラム追加 |
| `docs/er-diagram.md` | MVP結果テーブルのカラム定義更新 |

### 2. タイマー機能

| 項目 | 内容 |
|------|------|
| 必要度 | ★★★★☆ |
| 実装難易度 | 低 |
| カテゴリ | 試合・プレイ体験 |
| 実装順 | 2番目 |
| ステータス | 実装済み |

**概要**: 進行中マッチのUI内にタイマーボタンを配置し、手動でカウントダウンタイマーを開始できるようにする。

#### 方針

- クライアントサイドのみで完結（`setInterval` ベース）。サーバー連携不要
- マッチ単位のタイマー。進行中マッチのUI内に配置
- 手動開始（試合開始とタイマー開始を分離し、現場の柔軟性に対応）

#### タイマー仕様

| 項目 | 内容 |
|------|------|
| 時間設定 | セレクトボックス（1〜20分） |
| デフォルト | 5分 |
| 最大 | 20分 |
| 前後半 | なし（カウントダウン1本） |

#### UI配置

- 進行中マッチの「ホイッスル！」ボタンの横に `[⏱ タイマー]` ボタンを追加
- タップすると時間入力 → 開始
- カウントダウン中はマッチカード内に残り時間を表示
- 終了時にアラート

#### アラート

- **ビープ音**: Web Audio API（`AudioContext` + `OscillatorNode`）でブラウザ内生成。外部ファイル不要
- **振動**: `navigator.vibrate` を併用（Android対応、iOSは非対応）
- タイマー開始時のタップで `AudioContext` を初期化（スマホブラウザの自動再生制約への対応）

#### 注意事項

- バックグラウンドタブでは `setInterval` の精度が落ちるため、開始時刻からの経過時間で残り時間を計算する方式にする
- タイマーはクライアントローカルのため、他の端末とは同期しない

#### 影響範囲

| ファイル | 変更内容 |
|----------|----------|
| `src/js-rounds.html` | タイマーロジック（開始・カウントダウン・アラート）、進行中マッチUIへのボタン追加 |
| `src/css.html` | タイマー表示・ボタンのスタイル追加 |

### 3. ハイライトコメント生成

| 項目 | 内容 |
|------|------|
| 必要度 | ★★★☆☆ |
| 実装難易度 | 中 |
| カテゴリ | ソーシャル・エンタメ |
| 実装順 | 3番目 |
| ステータス | 未着手 |

**概要**: 試合終了時にGemini APIでスポーツニュース風のハイライト記事を1試合ごとに自動生成する。終了したマッチカードをタップすると記事が読める。

#### フロー

```
「ホイッスル！」タップ
  → コメント入力ダイアログ（任意、スキップ可）
  → サーバーに送信（試合終了処理 + Gemini API呼び出し）
  → ローディング表示（「ハイライト生成中...📝」）
  → ハイライト記事をマッチシートに保存
  → 終了したマッチカードをタップするとハイライト記事が展開表示される
```

#### 仕様

| 項目 | 内容 |
|------|------|
| 生成単位 | 1試合（マッチ）ごと |
| 生成タイミング | 「ホイッスル！」ボタン押下時（同期処理） |
| 入力データ | マッチの得点・スコア・メンバー情報 + 操作者のコメント（任意） |
| 出力 | スポーツニュース風のハイライト記事（テキスト） |
| 保存先 | マッチシートに「ハイライト」カラムを追加 |

#### コメント入力

- 「ホイッスル！」タップ後にダイアログを表示
- 操作者（主に管理者）が試合の感想・印象を自由入力
- 任意入力（スキップ可）。入力があればGeminiのプロンプトに含める
- 現場の空気感をハイライトに反映する目的

#### 表示UI

- 終了したマッチカードをタップするとアコーディオン展開でハイライト記事を表示
- ハイライトが生成済みのマッチカードには目印（📰アイコン等）を表示

#### Geminiプロンプト方針

- **モデル: Gemini 3.1 Flash-Lite**（安価なモデルを使用。試合ごとに呼び出すためコストを抑える）
- スポーツ記者トーンで、エンタメ性のある記事を生成
- 「○○選手がハットトリック達成！」のような盛り上がる表現
- 堅すぎず、フットサルの楽しい雰囲気に合ったトーン
- 操作者のコメントがある場合は、現場の雰囲気として考慮

#### 注意事項

- Gemini API呼び出し分、ホイッスルから試合終了完了まで数秒の待ち時間が発生する
- ローディング表示で体験をカバー
- API呼び出し失敗時は試合終了処理のみ完了させ、ハイライトは未生成のままにする（試合終了を妨げない）

#### 影響範囲

| ファイル | 変更内容 |
|----------|----------|
| `src/js-rounds.html` | ホイッスル処理の拡張（コメント入力ダイアログ、ハイライト表示UI） |
| `src/Rounds.gs` | `endMatch` の拡張（コメント受け取り、Gemini呼び出し、ハイライト保存） |
| `src/Code.gs` | マッチシートのヘッダーに `ハイライト`・`終了コメント` カラム追加 |
| `src/css.html` | ハイライト表示・アコーディオン展開のスタイル追加 |
| `docs/er-diagram.md` | マッチテーブルのカラム定義更新 |

## 推奨実装順の根拠

1. **得点ランキング / 勝ち点ランキング / スタッツ**: 既存UIの拡張で完結し、最も低コストで効果が高い
2. **タイマー**: クライアントサイドのみで独立して実装可能。試合体験に直結するため早めに
3. **ハイライト生成**: Gemini連携は既存のため比較的容易。エンタメ性が高い
