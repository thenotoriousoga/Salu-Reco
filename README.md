# Futsal Manager

フットサルの試合管理・MVP選出を行うWebアプリです。
Google Apps Script（GAS）で完結し、スマホブラウザから利用できます。

## データ構造

全てのデータは **イベント単位** で管理されます。イベントを横断した集計はありません。

```
イベント（例: 5/1 フットサル）
├── メンバー（このイベントの参加者 15人）
│   └── 名前 / 年次 / サッカー経験 / 幹事フラグ
├── ラウンド（第1試合、第2試合...）
│   ├── チーム分け（AI or ランダム）
│   ├── スコア
│   └── 得点者
├── アンケート（Googleフォーム自動生成）
│   └── 各メンバーへのコメント（任意）
├── MVP結果（定量+定性評価）
└── イベント内成績（得点・勝敗）
```

## 機能一覧

### イベント実施前

| 機能 | 説明 |
|---|---|
| イベント作成 | 日付と名称を入力して作成 |
| メンバー登録 | 名前・年次・サッカー経験・幹事フラグを入力し「リストに追加」→ プレビュー確認 →「まとめて登録」で一括保存。Enterキーで連続入力可。年次・経験の設定は前の人の値が残るので同属性の人を連続入力しやすい |

### 当日（フットサル中）

| 機能 | 説明 |
|---|---|
| チーム分け（AI） | サッカー経験・年次を考慮した蛇行ドラフト方式でバランスよく分配 |
| チーム分け（ランダム） | Fisher-Yatesシャッフルで完全ランダム分配 |
| 複数ラウンド | 第1試合、第2試合...と何回でもチーム分け・試合を追加可能。各ラウンドのチーム構成が記録される |
| 試合記録 | 得点者ボタンをタップするとスコアが自動計算される。手動でのスコア±1も可能 |

### 試合後

| 機能 | 説明 |
|---|---|
| アンケート | Googleフォームを自動生成。各メンバーへのコメントを任意で記入してもらう（MVPが誰かは書かない） |
| 回答取得 | フォームの回答をスプレッドシートに取り込み |
| MVP選出 | 定量評価（得点・勝敗・経験補正・年次補正）+ 定性評価（コメント分析）で自動選出 |
| MVP/準MVP設定 | それぞれの人数を1〜5人で設定可能 |
| イベント内成績 | 得点数・勝敗を集計して表示 |

### MVP選出ロジック

| 評価軸 | 配点 | 補正 |
|---|---|---|
| 定量評価（50%） | 得点×3pt、勝利×2pt | 未経験者+30%、若手（2年目以下）+15% |
| 定性評価（50%） | コメント数×2pt、ポジティブワード+1pt | 長文（30字超）+1pt、（60字超）+2pt |

両方を0-50に正規化して合算（100点満点）。

## セットアップ

### 1. スプレッドシートを用意

[Google スプレッドシート](https://sheets.google.com) で新規作成し、「拡張機能」→「Apps Script」を開きます。

### 2. ファイルをコピー

Apps Script エディタで以下のファイルを作成し、`src/` 内の内容をコピーしてください。

| Apps Script ファイル名 | コピー元 |
|---|---|
| `Code.gs` | `src/Code.gs` |
| `Events.gs` | `src/Events.gs` |
| `Members.gs` | `src/Members.gs` |
| `Rounds.gs` | `src/Rounds.gs` |
| `Survey.gs` | `src/Survey.gs` |
| `Mvp.gs` | `src/Mvp.gs` |
| `index.html` | `src/index.html` |
| `css.html` | `src/css.html` |
| `js.html` | `src/js.html` |

### 3. スプレッドシートIDを設定

コンテナバインドスクリプト（スプレッドシートからApps Scriptを開いた場合）はこの手順は不要です。

スタンドアロンスクリプトの場合：
1. Apps Script エディタ →「プロジェクトの設定」→「スクリプトプロパティ」
2. プロパティ `SPREADSHEET_ID`、値にスプレッドシートのIDを設定

### 4. シートを初期化

1. `Code.gs` を開く
2. 関数セレクタで `initializeSheets` を選択 → ▶ 実行
3. 初回は権限の承認が求められるので許可

7つのシート（イベント、メンバー、ラウンド、ラウンドメンバー、得点、アンケート回答、MVP結果）が作成されます。

### 5. Webアプリとしてデプロイ

1. 「デプロイ」→「新しいデプロイ」→ 種類「ウェブアプリ」
2. 次のユーザーとして実行：**自分**
3. アクセスできるユーザー：**全員**
4. URLをコピーして共有

## 使い方の流れ

```
1. イベント作成（日付・名称）
2. メンバー登録（名前・年次・経験・幹事 → リストに追加 → まとめて登録）
3. チーム分け（AI分け or ランダム → プレビュー確認 → 試合開始）
4. 試合記録（得点者をタップ → スコア自動計算）
5. 試合終了 → 次のラウンドを追加 or 全試合終了
6. アンケートフォーム作成 → URL共有 → 回答取得
7. MVP選出実行 → 結果表示
```

## 画面構成

- **ホーム** — 最近のイベント一覧
- **イベント** — イベント作成・一覧
  - **メンバータブ** — メンバー登録（キュー方式で一括登録）
  - **試合タブ** — チーム分け・ラウンド管理・スコア記録
  - **結果タブ** — イベント内成績・アンケート・MVP選出

## ファイル構成

```
src/
├── Code.gs          共通処理（スプレッドシート取得、シート初期化、ユーティリティ）
├── Events.gs        イベントCRUD
├── Members.gs       メンバーCRUD（イベントに紐づく）
├── Rounds.gs        ラウンド管理、チーム分け（AI/ランダム）、スコア・得点記録
├── Survey.gs        Googleフォーム自動生成、回答取得
├── Mvp.gs           MVP選出ロジック（定量+定性評価）
├── index.html       メインHTML
├── css.html         スタイルシート（Fira Sans / Fira Code、スマホ最適化）
└── js.html          クライアントサイドJS
```

## スプレッドシート構成

| シート名 | カラム |
|---|---|
| イベント | イベントID / 日付 / 名称 / ステータス / MVP人数 / 準MVP人数 / フォームURL / フォームID |
| メンバー | メンバーID / イベントID / 名前 / 年次 / サッカー経験 / 幹事 |
| ラウンド | ラウンドID / イベントID / ラウンド番号 / チームA名 / チームB名 / スコアA / スコアB / ステータス |
| ラウンドメンバー | ラウンドID / メンバーID / チーム |
| 得点 | ラウンドID / メンバーID / 得点数 |
| アンケート回答 | イベントID / 回答者名 / 対象メンバーID / 対象メンバー名 / コメント |
| MVP結果 | イベントID / メンバーID / 名前 / 順位 / 理由 / 定量スコア / 定性スコア / 総合スコア |

## 技術構成

- **バックエンド**: Google Apps Script（6ファイル）
- **フロントエンド**: HTML + CSS + Vanilla JS（GASテンプレート）
- **データ**: Google スプレッドシート（7シート）
- **アンケート**: Google フォーム（自動生成）
- **フォント**: Fira Sans（本文）/ Fira Code（データ表示）
- **デザイン**: Vibrant & Block-based スタイル、SVGアイコン、スマホファースト
