# Salu-Rec

フットサルの試合管理・MVP選出を行うWebアプリです。
Google Apps Script（GAS）で完結し、スマホブラウザから利用できます。

## データ構造

全てのデータは **イベント単位** で管理されます。イベントを横断した集計はありません。

```
イベント（例: 5/1 フットサル）
├── メンバー（このイベントの参加者 15人）
│   └── 名前 / 年次 / サッカー経験 / 幹事フラグ / 備考
├── ラウンド（第1試合、第2試合...）
│   ├── チーム分け（AI or ランダム、2〜5チーム対応）
│   ├── スコア
│   └── 得点者
├── アンケート（Googleフォーム自動生成）
│   └── 各メンバーへのコメント（任意）
├── MVP結果（定量+定性評価）
└── イベント内成績（得点・勝敗）
```

## ロール（権限）

アプリには **管理者** と **一般ユーザー** の2つのロールがあります。

| 機能 | 管理者 | 一般ユーザー |
|---|:---:|:---:|
| イベント作成・削除 | ○ | × |
| イベント設定変更 | ○ | × |
| メンバー登録・編集・削除 | ○ | × |
| メンバー一覧の閲覧 | ○ | ○ |
| ラウンド作成（チーム分け） | ○ | ○ |
| マッチのスコア・得点記録 | ○ | ○ |
| ラウンド終了 | ○ | ○ |
| ラウンド削除 | ○ | × |
| 成績閲覧 | ○ | ○ |
| アンケートフォーム作成・回答取得 | ○ | × |
| アンケートフォームを開く | ○ | ○ |
| MVP選出の実行 | ○ | × |
| MVP結果の閲覧 | ○ | ○ |

- **管理者**: パスワードでログイン。全機能にアクセス可能
- **一般ユーザー**: イベントコード（4桁英数字）を入力して参加。試合管理と閲覧が可能

## 機能一覧

### イベント実施前

| 機能 | 説明 |
|---|---|
| イベント作成 | 日付と名称を入力して作成 |
| メンバー登録 | 名前・年次・サッカー経験・幹事フラグを入力し「リストに追加」→ プレビュー確認 →「まとめて登録」で一括保存。Enterキーで連続入力可。年次・経験の設定は前の人の値が残るので同属性の人を連続入力しやすい |

### 当日（フットサル中）

| 機能 | 説明 |
|---|---|
| チーム分け（AI） | サッカー経験・年次を考慮した蛇行ドラフト方式でNチームにバランスよく分配。チーム数は2〜最大（人数÷3）まで設定可能。3チーム以上の場合は対戦カードをユーザーが選択 |
| チーム分け（ランダム） | Fisher-Yatesシャッフル後、ラウンドロビンでNチームに分配 |
| 複数ラウンド | 第1試合、第2試合...と何回でもチーム分け・試合を追加可能。各ラウンドのチーム構成が記録される |
| 試合記録 | 得点者ボタンをタップするとスコアが自動計算される。手動でのスコア±1も可能 |

### 試合後

| 機能 | 説明 |
|---|---|
| アンケート | Googleフォームを自動生成。各メンバーへのコメントを任意で記入してもらう（MVPが誰かは書かない） |
| 回答取得 | フォームの回答をスプレッドシートに取り込み |
| MVP選出 | 試合データ・メンバー情報・アンケートコメントをGemini AIが総合評価して自動選出 |
| MVP/準MVP設定 | それぞれの人数を1〜5人で設定可能 |
| イベント内成績 | 得点数・勝敗を集計して表示 |

### MVP選出ロジック

Gemini AIによる総合評価（0〜100点）で選出します。
詳細は [MVP選出ロジック](docs/mvp-logic.md) を参照してください。

試合データ（得点・勝利）、メンバー情報（経験・年次・備考・幹事）、アンケートコメントを全てAIに渡し、現場の雰囲気に合ったMVPを選出します。得点が多い＝MVPではなく、場の盛り上げやチームメイトからの評価を重視します。

## セットアップ

### 1. スプレッドシートを用意

[Google スプレッドシート](https://sheets.google.com) で新規作成し、「拡張機能」→「Apps Script」を開きます。

### 2. ファイルをコピー

Apps Script エディタで以下のファイルを作成し、`src/` 内の内容をコピーしてください。

| Apps Script ファイル名 | コピー元 |
|---|---|
| `Code.gs` | `src/Code.gs` |
| `Events.gs` | `src/Events.gs` |
| `Members.gs` | `src/Members.gs` |
| `Rounds.gs` | `src/Rounds.gs` |
| `Survey.gs` | `src/Survey.gs` |
| `Mvp.gs` | `src/Mvp.gs` |
| `index.html` | `src/index.html` |
| `css.html` | `src/css.html` |
| `js.html` | `src/js.html` |

### 3. スプレッドシートIDを設定

コンテナバインドスクリプト（スプレッドシートからApps Scriptを開いた場合）はこの手順は不要です。

スタンドアロンスクリプトの場合：
1. Apps Script エディタ →「プロジェクトの設定」→「スクリプトプロパティ」
2. プロパティ `SPREADSHEET_ID`、値にスプレッドシートのIDを設定

### 4. シートを初期化

1. `Code.gs` を開く
2. 関数セレクタで `initializeSheets` を選択 → ▶ 実行
3. 初回は権限の承認が求められるので許可

8つのシート（イベント、メンバー、ラウンド、マッチ、マッチメンバー、得点、アンケート回答、MVP結果）が作成されます。

### 5. 管理者パスワードを設定

1. Apps Script エディタ →「プロジェクトの設定」→「スクリプトプロパティ」
2. プロパティ `ADMIN_PASSWORD`、値に任意のパスワードを設定

### 6. Webアプリとしてデプロイ

1. 「デプロイ」→「新しいデプロイ」→ 種類「ウェブアプリ」
2. 次のユーザーとして実行：**自分**
3. アクセスできるユーザー：**全員**
4. URLをコピーして共有

## 使い方の流れ

### 管理者
```
1. 管理者パスワードでログイン
2. イベント作成（日付・名称・参加コード） → 参加コードをメンバーに共有
3. メンバー登録（名前・年次・経験・幹事 → リストに追加 → まとめて登録）
4. チーム分け（チーム数選択 → AI分け or ランダム → プレビュー確認 → ラウンド作成）
6. 試合追加（ラウンド内で対戦カードを選択 → 試合開始）
7. 試合記録（得点者をタップ → スコア自動計算）
8. 試合終了 → 同じラウンドで次の試合を追加 or 新しいラウンドを作成
9. アンケートフォーム作成 → URL共有 → 回答取得
10. MVP選出実行 → 結果表示
```

### 一般ユーザー（参加者）
```
1. 管理者から共有された参加コードを入力
2. イベント詳細を閲覧（メンバー・試合結果・成績・MVP結果）
3. 試合管理（チーム分け・スコア記録・得点記録）も可能
4. アンケートフォームから回答
```

## 画面構成

- **ログイン** — 管理者パスワード or イベントコードで認証
- **イベント一覧**（管理者のみ） — イベント作成・一覧
  - **メンバータブ** — メンバー登録（管理者のみ）・一覧表示
  - **試合タブ** — チーム分け・ラウンド管理・スコア記録
  - **結果タブ** — イベント内成績・MVP選出（管理者のみ）・結果閲覧

## ファイル構成

```
src/
├── Code.gs          共通処理（スプレッドシート取得、シート初期化、ユーティリティ）
├── Events.gs        イベントCRUD
├── Members.gs       メンバーCRUD（イベントに紐づく）
├── Rounds.gs        ラウンド管理、チーム分け（AI/ランダム）、スコア・得点記録
├── Survey.gs        Googleフォーム自動生成、回答取得
├── Mvp.gs           MVP選出ロジック（定量+定性評価）
├── index.html       メインHTML
├── css.html         スタイルシート（Fira Sans / Fira Code、スマホ最適化）
└── js.html          クライアントサイドJS
```

## スプレッドシート構成

| シート名 | カラム |
|---|---|
| イベント | イベントID / 日付 / 名称 / ステータス / フォームURL / フォームID / コード |
| メンバー | メンバーID / イベントID / 名前 / 年次 / サッカー経験 / 幹事 / 備考 |
| ラウンド | ラウンドID / イベントID / ラウンド番号 / チーム分けJSON / ステータス |
| マッチ | マッチID / ラウンドID / マッチ番号 / チームA名 / チームB名 / スコアA / スコアB / ステータス |
| マッチメンバー | マッチID / メンバーID / チーム |
| 得点 | マッチID / メンバーID / 得点数 |
| アンケート回答 | イベントID / 回答者名 / 対象メンバーID / 対象メンバー名 / コメント |
| MVP結果 | イベントID / メンバーID / 名前 / 順位 / 理由 / 総合スコア |

テーブル間のリレーションやカスケード削除の詳細は [ER図](docs/er-diagram.md) を参照してください。

UIのデザインシステム（カラーパレット、タイポグラフィ、コンポーネント仕様、アクセシビリティ対応など）は [デザインシステム](docs/design-system.md) を参照してください。

## 技術構成

- **バックエンド**: Google Apps Script（6ファイル）
- **フロントエンド**: HTML + CSS + Vanilla JS（GASテンプレート）
- **データ**: Google スプレッドシート（8シート）
- **アンケート**: Google フォーム（自動生成）
- **フォント**: Fira Sans（本文）/ Fira Code（データ表示）
- **デザイン**: Vibrant & Block-based スタイル、SVGアイコン、スマホファースト
